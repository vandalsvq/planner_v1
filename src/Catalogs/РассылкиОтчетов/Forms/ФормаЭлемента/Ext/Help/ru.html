Рассылки отчетов предназначены для автоматического формирования отчетов и доставки сформированных отчетов получателям.

Реквизиты:

  • Наименование
    		•  представление рассылки в списках, может быть указано как шаблон.
  • Подготовлена
    		•  флаг готовности рассылки к выполнению. Включение активизирует дополнительные проверки заполнения реквизитов.
  • Отправлять
    		•  вид рассылки, определяющий цель:

    Отчеты указанным получателям - все способы доставки (по электронной почте и публикация).
    Свой отчет для каждого получателя - только по электронной почте, но доступны параметры персонализации.
    Только мне (личная рассылка автора) - только по электронной почте, заполняется минимальное количество полей.

  • Получатели
    		•  тип получателей рассылки, для которых будут отправлены отчеты.
  • Почтовый адрес
    		•  вид почтового адреса получателей.
  • Отчет и настройки
    		•  вариант отчета или дополнительный отчет и пользовательские настройки.

    Страница "Расписание":

  • Периодичность расписания 
    		•  определяет частотность срабатывания рассылки. Переключение данного параметра влияет на все настройки расписания.

    Ежедневно - рассылка будет выполняться с заданной частотой в днях.
    Еженедельно - по выбранным дням недели.
    Ежемесячно - по выбранным месяцам в требуемый день месяца.
    Расширенные параметры - позволяет детально настроить каждый параметр расписания.

  • Время начала
    		•  время запуска рассылки.
    При запуске по расписанию рассылка выполняется от имени автора.

    Страница "Доставка":

    Содержимое страницы может меняться, в зависимости от вида рассылки.

  • Публиковать
    		•  признак необходимости публикации рассылки.

    В папку - отчеты будут сохранены в указанную папку подсистемы "Работа с файлами".
    В сетевой каталог - отчеты будут сохранены в каталог файловой системы. Запись производится с сервера, поэтому рекомендуется указывать сетевой путь в формате UNC, например: \\server\folder\.
    На FTP ресурс - отчеты будут сохранены на ftp сервере.

    Подробнее - см. работа в клиент-серверном режиме.

  • Проверить
    		•  проверка публикации путем записи и удаления тестового файла. Тест считается успешным если ошибок не возникло.
  • Отправлять по электронной почте
    		•  признак использования электронной почты для отправки отчетов (или уведомлений).
  • От
    		•  учетная запись для отправки.
  • Проверить электронную почту
    		•  проверка учетной записи электронной почты путем отправки тестового сообщения. Тест считается успешным если ошибок не возникло.
  • Включить (исключить) в рассылку
    		•  список получателей и исключений из получателей рассылки. Можно выбирать элементы или группы получателей.
  • Тема и текст письма
    		•  могут содержать шаблоны параметров рассылки:
    [ДатаВыполнения] - дата рассылки. Параметр может быть указан в имени архива.
    [Автор] - пользователь, запустивший рассылку.
    [Получатель] - представление получателя рассылки. Доступен только для вида рассылки свой отчет для каждого получателя.
    [НаименованиеРассылки] - наименование рассылки. Параметр может быть указан в имени архива.
    [СформированныеОтчеты] - список отчетов и форматов, в которые они были сохранены.
    [ЗаголовокСистемы] - заголовок информационной базы 1С:Предприятие для всех пользователей.
    [СпособДоставки] - представление ресурса, на котором был опубликован отчет.

    Страница "Дополнительно":

  • Форматы сохранения отчетов
    		•  определяет форматы, в которых будут сохранены отчеты. Для каждого отчета можно задать свой набор форматов.
  • Отправлять, если пустой
    		•  признак отправки сформированного отчета даже если он пустой. Отчеты с графикой всегда считаются не пустыми.
  • Архивировать в ZIP
    		•  признак архивации сформированных отчетов. Рекомендуется включать архивацию если отчеты отправляются по электронной почте.
  • Адрес ответа
    		•  E-mail адрес, на который будут высылаться ответы на рассылку.

    Команды:

  • Выполнить сейчас
    		•  Позволят выполнить выбранные рассылки или группы рассылок от имени текущего пользователя. Выполняться будут только те рассылки, у которых установлен флажок подготовленности.
  • События рассылки
    		•  Позволяет открыть журнал регистрации, предварительно отфильтровав его по событиям рассылки со времени последнего запуска.

    Работа в клиент-серверном режиме:

    При работе информационной базы в клиент-серверном режиме необходимо иметь в виду, что рассылка выполняется на сервере 1С:Предприятия.

    Путь к одному ресурсу может отличаться для клиента и сервера, в зависимости от их сетевого расположения и операционных систем.

    Если сервер 1С:Предприятия работает под Windows, то необходимо убедиться, что сетевой каталог Windows доступен для чтения и записи пользователю операционной системы, от имени которого запущен сервер 1С:Предприятия. Если это не так, то необходимо дать этому пользователю необходимые права, или выбрать другого пользователя для запуска сервера 1С:Предприятия с необходимыми правами к сетевому каталогу. Эти же рекомендации актуальны и для доступа к ресурсам FTP и электронной почты.

    Поле сетевой каталог Linux предназначено для серверов 1С:Предприятия работающих под управлением ОС Linux. В этом поле можно указать только локальный путь, например /home/path. Чтобы использовать UNC путь, нужно сделать на него ссылку. Для этого под правами администратора (root) сделайте следующее:

    mkdir folder
    --- вместо folder напишите имя каталога. например в каталоге /home делаем mkdir shared - создаст каталог shared с полным путем /home/shared.

    mount -t cifs //server/share folder -o //domain/user, password=pwd
    - где //server/share - UNC путь к другому компьютеру , folder - путь каталога на Linux компьютере, например \home\shared, //domain/user - имя домена/пользователя, если нужно, pwd - пароль пользователя (если
    нужно).

    Если имя и пароль не нужны, команда выглядит так:
    mount -t cifs //server/share folder

    После этого в пути к тому указываем folder, например /home/shared

    Подробнее об администрировании сервера 1С:Предприятия см. документацию к платформе 1С:Предприятие.
